### Key Issue Summary

The most significant rule violation is on **lines 17 and 19**: assigning a `ReadOnlyListWrapper` directly to a `ReadOnlyListProperty` field bypasses the intended read-only pattern. The correct idiom is:

```java
private final ReadOnlyListWrapper<String> headerFormatList =
        new ReadOnlyListWrapper<>(...);
// expose as:
public ReadOnlyListProperty<String> headerFormatListProperty() {
    return headerFormatList.getReadOnlyProperty();
}
```
